Sujet : La telephonie sur IP (VoIP) en entreprise

BTS SIO SISR — Sujet de recherche
La telephonie sur IP (VoIP)

Au cours de mon stage chez 2iT Solutions, j'ai configure et deploye de nombreux equipements VoIP : telephones SIP Yealink, bases DECT, firewalls avec VLANs dedies. Ce sujet illustre directement comment les reseaux et la telephonie convergent dans un environnement professionnel moderne.

📡
Qu'est-ce que la VoIP ?

La Voice over IP (VoIP) transporte la voix sous forme de paquets IP en remplacement des lignes analogiques. Le protocole SIP (Session Initiation Protocol) est le standard pour etablir, maintenir et terminer les appels. L'audio est encode en temps reel via le protocole RTP.

SIPRTPH.323
🔧
Infrastructure reseau necessaire

La VoIP est sensible a la latence et aux pertes de paquets. Un VLAN dedie a la voix est indispensable pour isoler et prioriser ce trafic (QoS). Le firewall doit autoriser les flux SIP et RTP tout en les filtrant — c'est exactement ce que j'ai fait sur les Zyxel USG en stage.

VLAN voixQoSFirewall
☎️
Provisionnement centralise

Les telephones IP modernes (Yealink T35W, T85W) se configurent via un serveur d'approvisionnement. L'appareil recupere sa configuration au demarrage : compte SIP, serveur, parametres reseau. Cela permet de deployer des dizaines de postes rapidement sans intervention manuelle sur chacun.

YealinkProvisioningSIP account
📶
DECT IP : la mobilite en entreprise

Le DECT IP (base W70B + combines W73H) etend la VoIP sans fil. La base s'enregistre sur le serveur SIP et les combines s'appairent a la base via un code PIN DECT. Le firmware se met a jour automatiquement via la base — ce que j'ai realise et teste en semaine 5 de stage.

W70BW73HDECT
🔒
Securite des flux VoIP

Le trafic SIP peut etre intercepte ou usurpe (SIP spoofing, VLAN hopping). Les bonnes pratiques : utiliser TLS pour la signalisation, SRTP pour le chiffrement de la voix, cloisonner le VLAN voix. En stage, j'ai desactive la verification TLS pour les certificats auto-signes du serveur client.
